    10.1. 特征一致性挑战
    训练-服务偏差问题：在机器学习项目中，80%以上的模型性能下降源于特征不一致——离线训练使用的特征逻辑与线上推理不匹配，导致模型效果大幅衰减。
    特征口径漂移：常见于推荐系统、风控模型等场景，当数据团队和算法团队使用不同SQL计算特征时，模型线上表现可能直接"跳水"。
    解决方案：Feast通过"特征定义一次，离线在线用同一套逻辑" 的核心理念，彻底消除特征口径差异。
    10.2. 核心价值主张
    一致性保证：确保训练与推理使用完全相同的特征逻辑，避免模型效果衰减。
